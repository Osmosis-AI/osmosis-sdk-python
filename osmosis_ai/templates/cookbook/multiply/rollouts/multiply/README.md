# Multiply rollout

Cookbook starter rollout for an agent that calls a single `multiply` tool
and emits its final answer in `#### <number>` form. Generated by:

    osmosis template apply multiply

`osmosis template apply` writes this rollout directly into
`rollouts/multiply/` (and the matching configs into
`configs/training/multiply.toml` / `configs/eval/multiply.toml`). Each
rollout under `rollouts/<name>/` is self-describing: an entrypoint file
(`main.py`), a dependency manifest (`pyproject.toml`), and this
`README.md`.

## Layout

    rollouts/multiply/
    ├── main.py                 # Entrypoint for `osmosis eval run` / `osmosis train submit`.
    ├── pyproject.toml          # Rollout-local dependencies.
    ├── README.md               # This file.
    └── multiply_rollout/
        ├── __init__.py
        ├── workflow.py         # MultiplyWorkflow + multiply_workflow_config
        ├── grader.py           # MultiplyGrader + multiply_grader_config
        ├── tools.py            # `multiply(a, b)` Strands tool
        └── utils.py            # `extract_solution()` regex helper

## Run it

Step 1 — install the rollout's deps from the project root:

    pip install -e rollouts/multiply

Step 2 — run an end-to-end eval smoke test:

    osmosis --json eval run configs/eval/multiply.toml --limit 1

Step 3 — run a local eval against a JSONL dataset:

    osmosis --json eval run configs/eval/multiply.toml

Step 4 — commit and push the rollout code from the Platform-created repository:

    git push

Training submissions fetch code from the repository identified by the checkout's
`origin` remote, so pushed code is required.

Step 5 — submit a managed training run:

    osmosis --json train submit configs/training/multiply.toml

The eval and training configs (`configs/eval/multiply.toml`,
`configs/training/multiply.toml`) reference this rollout via
`rollout = "multiply"` / `entrypoint = "main.py"`.

For ad-hoc server testing, you can still run `python rollouts/multiply/main.py`;
the template workflow above uses eval and managed training commands.

## Customizing

- **Tool surface** — edit `multiply_rollout/tools.py`.
- **Termination** — `MultiplyWorkflow.check_done` decides when the agent has
  produced a final answer.
- **Reward** — `MultiplyGrader.compute_reward` compares the extracted answer
  to `ctx.label` (the dataset's ground-truth product).
- **Sampling params** — `multiply_workflow_config.model.params`
  (`temperature`, `top_p`, `max_tokens`).
- **Turn cap** — the `range(8)` loop in `workflow.py`.
